My Story

From English Literature
to Engineering Documentation

I started with a BA in English Literature in India — which taught me one thing clearly: clarity is everything. Not style for its own sake, but precision in service of the reader.

That conviction took me from WriteTurn Services in Kolkata to Visme's B2B SaaS platform in Maryland — where I spent three years building API references, SDK guides, and developer documentation for a product used by 10M+ users globally.

Then came Tesla. Two internship rotations at Gigafactory Nevada on the LFP Cell Manufacturing team, where documentation isn't abstract — it's the difference between a correctly executed procedure and an error on a production line. I cleared 190+ JIRA tickets, logged 241 Fisheye commits (most of any technical writer), and helped recover over 570 engineering hours.

Now I'm completing my MS in Digital Media and Communication at Northeastern (GPA 3.9), with a thesis comparing AI-powered SOP authoring tools using an original quality framework I built — the Documentation Quality Framework for Manufacturing (DQF-M).

My five-year goal is Technical Project Manager. I'm building toward it deliberately — one documentation system at a time.
